# 仿 QQWeb 即时聊天系统开发顺序文档

## 1. 当前版本定位

当前项目是课程作业展示级基础版，已经具备：

- Vue3 聊天界面
- Node.js + Express 后端
- WebSocket 点对点消息转发
- 文本、表情、图片消息
- 在线用户状态
- 浏览器 localStorage 本地历史消息

当前版本的主要限制：

- 没有真实登录注册
- 没有后端数据库
- 历史消息不能跨浏览器、跨电脑同步
- 离线用户收不到消息
- 图片使用 base64 传输，只适合演示
- 没有好友关系、最近会话、未读数、群聊等功能
- 没有部署、安全、鉴权能力

因此后续开发应优先补齐“消息可靠性”和“用户体系”，再扩展 QQWeb 风格功能。

## 2. 推荐开发顺序总览

推荐按以下阶段开发：

1. 后端消息存储
2. 登录注册与用户身份
3. 最近会话与好友列表
4. 离线消息
5. 未读消息与消息状态
6. 图片上传优化
7. 群聊功能
8. UI 与交互增强
9. 部署与安全加固

## 3. 第一阶段：后端消息存储

### 目标

让消息不只保存在浏览器 localStorage，而是由后端统一保存。这样刷新页面、换浏览器、换电脑后，仍然可以加载历史消息。

### 建议实现

课程作业可以先使用 JSON 文件存储：

- `server/data/messages.json`
- `server/data/users.json`
- `server/data/friends.json`

如果想更正式，可以使用 SQLite。

### 需要新增接口

- `GET /api/messages?userId=user1&targetId=user2`
  - 获取两个用户之间的历史消息。
- `POST /api/messages`
  - 保存消息。

### 前端调整

- 页面打开时，优先从后端接口加载历史消息。
- localStorage 可以继续保留，作为本地缓存。
- 收到消息后，同时保存到后端和 localStorage。

### 优先级

最高。因为它直接解决“历史消息不同步”的核心问题。

## 4. 第二阶段：登录注册与用户身份

### 目标

把当前“随便输入 userId 登录”的方式，改成真实用户系统。

### 建议实现

先做简单版：

- 用户注册：用户名、密码
- 用户登录：用户名、密码
- 后端保存用户信息
- 登录成功后返回用户信息

课程作业阶段可以先不做复杂 Token，也可以使用简单 Token。

正式一点可以使用：

- `bcryptjs` 加密密码
- `jsonwebtoken` 生成登录 Token

### 需要新增接口

- `POST /api/register`
- `POST /api/login`
- `GET /api/me`

### WebSocket 调整

当前连接方式：

```js
ws://localhost:3000?userId=user1
```

后续可以改成：

```js
ws://localhost:3000?token=xxx
```

后端根据 Token 判断真实用户身份，避免用户冒充别人。

### 优先级

高。真实聊天系统必须有身份认证。

## 5. 第三阶段：最近会话与好友列表

### 目标

让页面更像 QQWeb，不再只显示固定的 `user1`、`user2`、`user3`。

### 建议实现

新增两个概念：

- 好友列表：当前用户有哪些联系人。
- 最近会话：最近聊过的人，按最后一条消息时间排序。

### 需要新增接口

- `GET /api/friends?userId=user1`
- `POST /api/friends`
- `GET /api/conversations?userId=user1`

### 前端调整

- 左侧联系人列表改为从后端加载。
- 最近会话显示最后一条消息摘要。
- 最近会话显示最后消息时间。

### 优先级

高。它会显著提升页面完整度和展示效果。

## 6. 第四阶段：离线消息

### 目标

当接收方不在线时，消息仍然保存到后端。接收方下次登录后，可以看到离线期间收到的消息。

### 实现方式

发送消息时：

1. 后端先保存消息。
2. 如果接收方在线，立即通过 WebSocket 推送。
3. 如果接收方离线，不推送，但消息保留在后端。

接收方登录时：

1. 前端请求历史消息。
2. 后端返回所有未读或历史消息。
3. 前端显示出来。

### 需要新增字段

消息对象可以扩展：

```js
{
  id,
  from,
  to,
  type,
  content,
  timestamp,
  status,
  read
}
```

### 优先级

高。它解决“对方不在线就收不到”的问题。

## 7. 第五阶段：未读消息与消息状态

### 目标

让聊天系统更接近真实产品。

### 建议增加

- 未读消息数量
- 消息发送中
- 已发送
- 发送失败
- 已读 / 未读

### 前端表现

联系人列表中显示未读红点或数字。

消息气泡旁显示：

- 发送中
- 已发送
- 失败

### 后端支持

需要维护消息状态字段：

```js
status: 'sending' | 'sent' | 'failed'
read: true | false
```

### 优先级

中高。适合在历史消息和离线消息完成后再做。

## 8. 第六阶段：图片上传优化

### 目标

把当前 base64 图片传输改成文件上传。

当前 base64 方式的问题：

- 消息体很大
- WebSocket 传输压力大
- localStorage 容量容易爆
- 不适合真实图片聊天

### 建议实现

新增图片上传接口：

- `POST /api/upload`

流程：

1. 前端选择图片。
2. 通过 HTTP 上传到后端。
3. 后端保存到 `server/uploads/`。
4. 后端返回图片 URL。
5. WebSocket 只发送图片 URL。

消息内容示例：

```js
{
  type: 'image',
  content: 'http://localhost:3000/uploads/xxx.png'
}
```

### 优先级

中。课程演示可以继续使用 base64，但如果要做得更完整，建议升级。

## 9. 第七阶段：群聊功能

### 目标

支持多个用户在同一个群里聊天。

### 建议新增数据

- 群信息
- 群成员
- 群消息

### 消息结构扩展

当前私聊消息：

```js
{
  from,
  to,
  type,
  content
}
```

群聊消息可以增加：

```js
{
  from,
  roomId,
  chatType: 'group',
  type,
  content
}
```

### WebSocket 转发逻辑

私聊：

- 发给指定 `to` 用户。

群聊：

- 发给群里的所有在线成员。

### 优先级

中。建议在单聊稳定后再做。

## 10. 第八阶段：UI 与交互增强

### 目标

让界面更像 QQWeb，并提升课程展示效果。

### 可增加功能

- 搜索联系人
- 最近会话排序
- 消息时间分割线
- 消息右键菜单
- 图片预览
- 回车发送 / Ctrl + Enter 换行设置
- 连接断开提示
- 自动重连
- 空状态优化
- 用户头像
- 昵称

### 优先级

中。它不改变核心通信逻辑，但能显著提升观感。

## 11. 第九阶段：部署与安全加固

### 目标

让两台电脑、不同网络环境下都能访问同一个聊天系统。

### 部署方式

开发测试：

- 同一局域网使用主机 IP 访问。

公网演示：

- 前端部署到 Nginx、Vercel 或静态服务器。
- 后端部署到云服务器。
- WebSocket 使用 `wss://`。

### 需要处理

- CORS
- WebSocket 地址配置
- 环境变量
- HTTPS / WSS
- Token 鉴权
- 文件上传大小限制
- 图片类型校验

### 优先级

中低。如果只是本机课程展示，可以最后做。如果要两台电脑真实演示，则需要提前做。

## 12. 建议里程碑

### 里程碑 1：基础可演示版

包含：

- 点对点聊天
- 表情
- 图片 base64
- localStorage 历史
- 在线状态

当前项目已经达到这个阶段。

### 里程碑 2：增强课程版

建议新增：

- 后端 JSON 文件保存消息
- 登录注册
- 好友列表
- 最近会话
- 离线消息

这个阶段最适合作为课程作业升级版。

### 里程碑 3：完整展示版

建议新增：

- 未读消息
- 消息状态
- 图片上传接口
- 用户头像和昵称
- 自动重连
- 更完整的 QQWeb 风格界面

### 里程碑 4：接近真实产品版

建议新增：

- 数据库
- Token 鉴权
- 群聊
- 文件存储
- HTTPS / WSS 部署
- 安全校验

## 13. 最推荐的下一步

如果只选一个方向继续开发，最推荐先做：

1. 后端保存历史消息
2. 页面刷新后从后端加载历史消息
3. 离线消息

原因是这三项能直接解决当前版本最明显的问题：消息只存在本地、换浏览器不同步、对方离线收不到。
